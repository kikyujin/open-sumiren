## Phase 1: ノートブックが「ファイル」になる

固定ファイル名の単一ノートブックから、複数のノートブックを管理できる状態へ。
デスクトップ（React Flow）をホーム画面として導入。

| Step | 内容 | 詳細 |
|------|------|------|
| 1.1 | React Router 導入 | `/` → デスクトップ、`/notebook/{name}` → シート。ブラウザタブで複数ノート同時に開ける |
| 1.2 | 保存フォーマット拡張 | スナップショット + メタデータ（名前、LLMモデル、作成日時、更新日時）。.sumiren ファイル |
| 1.3 | デスクトップ画面 | React Flow でノートブックをノード表示。ダブルクリック（または Ctrl+クリックで新タブ）でシートを開く。エッジはまだなし |
| 1.4 | デスクトップメニュー | 背景右クリック → 新規ノートブック作成 |
| 1.5 | ノードメニュー | ノード右クリック → 名前変更、LLMモデル設定、削除 |
| 1.6 | xLLM 関数 | クラウド LLM（Claude API 等）対応。バックエンドに `/api/xllm` 追加、数式パーサーに `xLLM` 認識追加。APIキー管理はバックエンド側 |
| 1.7 | シートメニュー | シート画面上部。ノートブック名表示、保存（Ctrl+S）、▶ LLM実行 |
| 1.8 | OSS 公開準備 | LICENSE 選定（MIT or Apache 2.0）、README 整備、インストール手順、セットアップスクリプト |
| 1.9 | GitHub 公開 | `open-sumiren` リポジトリ作成、初回リリース |

### 判定基準
- デスクトップから複数のノートブックを作成・開く・保存できる
- ブラウザタブで複数ノートを同時編集できる
- LLM / xLLM の両方がセルから実行できる
- GitHub で公開されており、README の手順で誰でもセットアップできる

## Phase 2: ノートブックが「状態」を持つ

チェーン実行でセル間の依存関係を解決し、ヘルスと代表値でノートブックの状態を可視化する。

| Step | 内容 | 詳細 |
|------|------|------|
| 2.1 | チェーン実行 | LLM式の出力先を別のLLM式が参照するケースを自動解決。依存グラフのトポロジカルソートで実行順序を決定（旧 cell_engine.py の資産を活用） |
| 2.2 | 代表値セル設定 | ノートブック設定で「このセルが代表値」を指定。メタデータに `headline_cell` として保存 |
| 2.3 | ノード上に代表値表示 | デスクトップのノードにノートブック名 + 代表値のプレビューを表示 |
| 2.4 | ヘルスチェック（バックエンド） | ファイルパスの存在確認 + セル状態（#UNREF, #UNSOLVED）の集計。API: `GET /api/health/{name}` |
| 2.5 | ノード上にヘルス表示 | 🟢🟡🔴 をノードに表示 |

### ヘルス判定基準（仮。後でチューニング）

| 状態 | 条件 |
|------|------|
| 🟢 | #UNREF = 0、#UNSOLVED = 0 |
| 🟡 | #UNSOLVED > 0（未実行のLLM式がある） |
| 🔴 | #UNREF > 0（壊れた参照がある） |

### 判定基準
- シート内でLLM式をチェーン実行できる（手動で順番に実行する必要がない）
- デスクトップを見るだけで各ノートブックの状態が把握できる

## Phase 3: ノートブックが「繋がる」

ノートブック間のセル参照を実装し、デスクトップ上でエッジとして可視化する。

| Step | 内容 | 詳細 |
|------|------|------|
| 3.1 | ノートブック間セル参照記法 | `=[xMBS.sumiren!B3]` で他ノートブックのセル値を読み取れる |
| 3.2 | 参照解決（バックエンド） | 参照先ノートブックをロードし、セル値を返す API |
| 3.3 | エッジ自動生成 + ノード位置永続化 | ノートブック間の参照関係を解析し、React Flow のエッジとして描画。双方向ナビゲーション対応。ノードの配置座標もデスクトップ設定として保存 |
| 3.4 | クロスノートブック LLM 実行 | LLM 式の input に他ノートブックのセルを指定可能 |

### 判定基準
- デスクトップ上でノートブック間の依存関係がエッジとして見える
- あるノートブックから別のノートブックのセル値を参照してLLM実行できる
